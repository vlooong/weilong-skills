# Patent Disclosure Drafting Guide

本技能提供专利交底书撰写的完整工作流程，包括文献调研、技术分析和文档撰写三个阶段。

---

## 工作流程

### 阶段一：文献调研（Literature Review）

**触发条件**：用户提供专利题目和内容描述时，首先进行文献调研。

#### 1.1 提取关键技术信息
根据用户的输入，提取以下信息：
- **技术领域**：确定所属的技术分类（如：人工智能、能源系统、通信技术等）
- **关键技术词**：提取 5-10 个核心技术术语
- **技术问题**：明确要解决的核心技术问题
- **技术方案**：理解主要的技术方法或创新点

#### 1.2 制定搜索策略
基于提取的信息，构建多维度的搜索查询：

**搜索维度**：
1. **现有技术方案搜索**：查找同类问题的现有解决方案
   - 查询模式：`{技术领域} + {技术问题} + 解决方案/方法`
   - 示例：`综合能源站 优化调度 算法`

2. **专利文献搜索**：查找相关专利，了解专利布局
   - 查询模式：`{技术问题} + 专利/发明专利`
   - 示例：`储能容量配置 专利`

3. **最新研究进展**：查找学术论文和技术报告
   - 查询模式：`{技术方案} + 研究/论文/最新`
   - 示例：`深度学习 能源预测 最新研究`

4. **技术难点搜索**：了解现有技术的不足和改进空间
   - 查询模式：`{技术问题} + 难点 挑战 局限性`

#### 1.3 执行文献搜索
使用 WebSearch 工具进行多轮搜索。

#### 1.4 文献分析整理
对搜索结果进行分析和分类整理，归纳现有技术的主要问题，明确本发明的创新点和改进方向。

### 阶段二：技术分析与确认

**触发条件**：文献调研完成后。

#### 2.1 确认技术方案
基于文献调研结果，与用户确认：
- 本发明的核心技术方案
- 与现有技术的区别
- 主要创新点和技术优势

#### 2.2 构建技术框架
- 确定数学符号体系
- 规划技术阶段划分

### 阶段三：交底书撰写

**触发条件**：技术方案确认后。

按照以下标准结构撰写交底书。

## 核心原则

### 1. 技术背景与创新点统一化
- **不要**在每个技术点单独叙述"技术背景与创新点"
- **应该**在开头（背景技术章节）统一阐述现有技术问题和本发明的创新点
- 技术背景分析应系统化：分为现有技术分类、存在问题、本发明的技术思路

### 2. 公式融入叙述，不单独列举
- **不要**在开头单独列举"核心公式框架"
- **应该**将公式融入到后续各个技术阶段的描述中
- 每个技术阶段根据需要引入相应的数学公式
- 公式引入后要立即解释参数含义和技术作用

### 3. 段落式叙述，减少列表
- **不要**过度使用分点列表（如"第一类...第二类..."、"一是...二是..."）
- **应该**采用段落形式进行连贯叙述
- 分类内容可以用段落中的转折连接词表达（如"此外"、"另外"）
- 仅在必要时（如枚举、并列项）使用列表

### 4. 技术步骤具体化，选择明确方案
- **不要**列举多个可选方法（如"可采用A或B或C"）
- **应该**确定一个具体的方案详细叙述
- 技术步骤要用数学公式的形式说明
- 算法要给出完整的计算过程，不能仅提及名称

### 5. 章节间衔接与连贯性
- 每个章节结尾要说明：本阶段完成了什么，为后续哪个阶段提供了什么基础
- 使用明确的数学符号体系，前后一致
- 技术叙述要具体，避免空洞描述

## 标准章节结构

### 前置部分
```yaml
---
title: 发明名称
date: YYYY-MM-DD
tags: [专利, 交底书, 相关技术标签]
status: draft
---
```

### 1. 技术领域
简要说明本发明涉及的技术领域，通常1-2句话。

### 2. 背景技术

#### 2.1 现有技术分析
采用段落式叙述，分类阐述现有技术。每类方法说明其工作原理、优点和局限。

**写作示例**：
```
目前，[技术领域]主要采用以下几类方法。第一类是基于[方法A名称]的方法。该方法[简要描述工作原理]。优点是[优点]，但局限在于[局限性]。第二类是基于[方法B名称]的方法。该方法[简要描述工作原理]。这类方法[优点/特点]，但存在以下问题：[问题]。
```

#### 2.2 现有技术存在的问题
采用段落式叙述，将多个问题整合为连贯的段落。

**写作示例**：
```
综合分析现有技术，主要存在以下若干方面的不足。一是[问题1]，[具体描述]。二是[问题2]，[具体描述]。三是[问题3]，[具体描述]。
```

#### 2.3 本发明的技术思路
- 将问题形式化描述
- 定义核心数学符号（但不要列出完整的公式框架）
- 提出建模思路和主要创新点

**写作示例**：
```
从数学建模角度，本发明将问题形式化如下。设[符号定义]...目标函数为[公式]。需要解决的核心问题包括[列举]。
```

### 3. 发明内容

#### 3.1 发明目的
简述本发明的核心目的，通常2-3个目的。

#### 3.2 技术方案
- 放置总体流程图（使用Mermaid语法）
- **数学符号定义**：定义基础符号，但不要列举完整的"核心公式框架"
- 说明整体技术方案的分阶段结构

**符号定义示例**：
```
为便于后续描述，定义本发明使用的核心数学符号体系如下。$V=\{v_1,...,v_n\}$表示[含义]，$E=\{e_1,...,e_m\}$表示[含义]...
```

#### 3.3 - 3.9 各技术阶段
每个阶段采用段落式叙述，包含：
- **阶段目标**：说明本阶段解决什么问题（第一段）
- **技术细节**：以段落形式详细描述，在适当时机引入公式
- **公式嵌入**：公式自然嵌入段落中，公式后立即解释参数
- **阶段总结**：最后一段说明本阶段产出，为下阶段提供什么

**阶段叙述示例**：
```
本阶段的核心目标是[目标描述]。将综合能源站的各类设备抽象为图节点[符号定义]。设备间的耦合关系分为[分类描述1]；[分类描述2]。对于每对设备$(v_i, v_j)$，计算耦合强度$W_{ij}$的公式为：

[公式]

其中[参数1]为[含义]，[参数2]为[含义]。耦合强度矩阵$W$形成[说明]。至此，阶段X完成了[总结]，为后续的[下一阶段]提供了[基础说明]。
```

**技术算法示例**（以联结树算法为例）：
```
本发明采用联结树算法进行精确推理。联结树算法的步骤如下：首先将DBN道德化，对于每个节点$v$，确保其所有父节点间存在边，形成道德图$\mathcal{G}^M$。然后对道德图进行三角化，通过添加边消除所有长度大于3的无弦环，得到三角化图$\mathcal{G}^T$。识别三角化图的所有最大团，构成团集合$\mathcal{C}=\{C_1, C_2, ..., C_K\}$。构建团图，选择团图的一棵生成树作为联结树$\mathcal{T}$。对于每个团$C_k$，计算团势函数[公式]。从叶子向根方向执行消息传递，对于每条边$(C_k, C_l)$（$C_k$为$C_l$的子节点），传递消息：

[公式]

其中[参数说明]。消息传递完成后，计算每个团的边缘分布：

[公式]

通过边缘分布的边缘化得到任意节点的后验概率分布。
```

### 3.10 本发明的有益效果
采用段落式叙述，系统总结本发明的优势（通常5-6点）。

**写作示例**：
```
基于上述N阶段技术方案，本发明具有以下显著有益效果。一是[效果1]，[具体描述]。二是[效果2]，[具体描述]。三是[效果3]，[具体描述]。
```

### 4. 具体实施方法
- 以具体案例说明实施过程
- 按阶段顺序描述
- 包含实际数据和参数

## 写作规范

### 数学公式规范
1. **公式要自成体系**：符号前后一致，有明确定义
2. **公式要嵌入叙述**：在技术描述中自然引入公式
3. **公式要有实际意义**：每个公式对应一个具体的技术问题
4. **公式后必解释**：公式后必须紧跟参数说明
5. **不要公式堆砌**：避免连续多个公式而无文字说明

### 语言规范
1. **使用简体中文**：包括术语和变量名说明
2. **技术叙述要具体**：避免空洞的"优化"、"改进"等词汇
3. **段落要衔接**：使用过渡句连接各段落
4. **减少列表使用**：优先使用段落叙述

### 图表规范
1. 流程图使用Mermaid语法
2. 图要有明确编号（如图3.1、图3.2）
3. 图要有详细说明（图下方用段落说明，不是列表）

## 常见错误避免

### ❌ 不要这样做

#### 错误1：单独列举核心公式框架
```
### 3.2.1 数学符号与建模框架

**核心公式框架**：

**公式1：能源站拓扑建模**
$G = (V, E, W), \quad W_{ij} = f_{\text{coup}}(v_i, v_j)$

**公式2：风险状态转移DBN模型**
$P(\mathbf{X}_t|\mathbf{X}_{t-1},\mathcal{G},\theta) = ...$
```

#### 错误2：过度使用列表
```
**第一类是基于阈值规则的单点报警方法。**
**第二类是基于层次分析法（AHP）...**
**第三类是基于动态贝叶斯网络（DBN）...**
```

#### 错误3：技术步骤列举多个选项
```
**精确推理**：对于小规模DBN，采用联结树算法或变量消元法。
**近似推理**：对于大规模DBN，采用似然加权或吉布斯采样。
```

#### 错误4：仅提及算法名称无具体步骤
```
本发明采用联结树算法进行精确推理。
```

### ✅ 应该这样做

#### 正确1：公式融入各阶段
```
本阶段的核心目标是构建能源站拓扑图。对于每对设备$(v_i, v_j)$，计算耦合强度$W_{ij}$的公式为：

[公式]

其中[参数说明]。
```

#### 正确2：段落式叙述
```
目前，综合能源站安全监测与应急响应主要采用以下几类方法。第一类是基于阈值规则的单点报警方法，该方法[描述]。第二类是基于层次分析法...该方法[描述]。第三类是基于动态贝叶斯网络...
```

#### 正确3：确定具体方案
```
对于中小规模问题，本发明采用单纯形法求解上述线性规划模型。将网络流优化问题转化为标准型[公式]...单纯形法从初始基本可行解开始，通过进基和出基操作迭代改进...
```

#### 正确4：算法步骤完整描述
```
本发明采用联结树算法进行精确推理。联结树算法的步骤如下：首先将DBN道德化[详细描述]。然后对道德图进行三角化[详细描述]。识别三角化图的所有最大团[公式]。构建团图[描述]...
```

## 特定章节写作模板

### 现有技术分析模板
```
目前，[技术领域]主要采用以下几类方法。第一类是基于[方法A名称]，该方法[原理描述]。优点是[优点]，但局限在于[局限性]。第二类是基于[方法B名称]，该方法[原理描述]。这类方法[优点/特点]，但存在[问题]。第三类是基于[方法C名称]，该方法[原理描述]。相关研究如[引用]证明[结论]，但存在[问题]。
```

### 现有问题总结模板
```
综合分析现有技术，主要存在以下若干方面的不足。一是[问题1标题]，[具体描述]和[影响]。二是[问题2标题]，[具体描述]。三是[问题3标题]，[具体描述]。
```

### 技术阶段叙述模板
```
本阶段的核心目标是[目标]。[技术背景和问题描述]。采用[具体方法]，具体实施如下：[步骤1详细描述，含公式]。[步骤2详细描述，含公式]。[步骤3详细描述，含公式]。至此，阶段X完成了[产出]，为后续的[下一阶段]提供了[基础]。
```

### 有益效果模板
```
基于上述N阶段技术方案，本发明具有以下显著有益效果。一是[效果1]，[具体说明]。二是[效果2]，[具体说明]。三是[效果3]，[具体说明]。四是[效果4]，[具体说明]。五是[效果5]，[具体说明]。六是[效果6]，[具体说明]。
```
