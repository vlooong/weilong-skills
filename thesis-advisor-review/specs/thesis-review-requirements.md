# Thesis Review Requirements (Master)

硕士学位论文审核的领域要求与边界条件。

## When to Use

| Phase | Usage | Section |
|-------|-------|---------|
| Phase 0 | 对齐审核目标与输入约定 | Requirements |
| Review Loop | 作为“是否必须修改”的裁决依据 | Requirements |

---

## Requirements

### 1) 输入约定（本 Skill 的默认配置）

- 输入为**单个 Markdown 文件**，标题层级采用 `#`（章）与 `##`（节）。
- 默认**按节推进**：每个 `##` 小节视为一个审核单元。
- 允许存在 `###`（子小节），在同一节内一并审核。

### 2) 结构与叙事

- 每章开头需要回答：本章解决什么问题、为什么需要、与全文贡献的关系。
- 每节开头需要给出“本节目标/要点”（1-3 句即可），避免读者迷失。
- 避免“本章安排如下”之类空话反复出现；出现一次即可。

### 3) 方法与论证（计算机类硕士常见硬门槛）

- 关键概念必须在首次出现处定义；符号与缩写必须一致。
- 核心方法必须包含：问题定义 → 设计动机 → 方案细节 → 复杂度/代价 → 适用边界。
- 任何“提升/优于/显著”等表述必须对应**可复现证据**（实验、消融、统计显著性或引用）。

### 4) 实验与结果（如论文包含实验）

- Baseline 必须公平：同数据划分、同评价指标、同训练资源（或解释差异）。
- 必须明确：数据集、预处理、超参、训练时长/硬件、随机种子（或重复次数）。
- 图表必须自洽：坐标轴/单位/误差棒（若适用）/图例解释完整。

### 5) 写作与表达

- 句子以信息密度为目标：避免一段话只表达一个显而易见事实。
- 术语统一（中文/英文/缩写）、图表编号引用一致。
- 结论不得引入正文未论证的新主张。

### 6) 本 Skill 的工作方式边界

- **不做**：替你“编造实验结果/数据”。
- **可以做**：提出需要补的实验、建议如何写实验设置、如何改写段落、如何降低审稿攻击面。

## Validation Function (informal)

- 若 `##` 小节缺少目标句、缺少与上一节/章节的承接、或关键主张无证据链 → 至少 REVISE。
